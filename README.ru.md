## Архитектура

### Основные компоненты

#### **1. App (`app/`)**
Центральный координатор, управляющий жизненным циклом приложения и оркестрировкой компонентов.

- **app.go** - Основная структура App со всеми подсистемами
- **init.go** - Инициализация и настройка всех компонентов
- **lifecycle.go** - Запуск, остановка и очистка приложения
- **interaction.go** - Обработка событий клавиатуры и команд пользователя
- **settings_flow.go** - Логика навигации меню настроек
- **tick.go** - Основной цикл событий и таймер кадров
- **install.go** - Утилиты установки
- **bootstrap/** - Процедуры загрузки
- **state/** - Управление состоянием приложения
- **events/** - Система событий и обработчики

#### **2. Движок анимации (`animation/`)**
Управляет таймингом кадров и контролем FPS для плавного визуального опыта.

- **engine.go** - Ограничение FPS и расчёты тайминга кадров
- Обеспечивает стабильный цикл анимации 60 FPS (настраивается)

#### **3. Система частиц (`rain/`)**
Физическое моделирование частиц, создающее эффект дождя.

- **types.go** - Структуры данных Particle и ParticleSystem
- **system.go** - Управление жизненным циклом частиц
- **spawn.go** - Логика появления (статическая, реактивная на музыку, направленная)
- **update.go** - Физическое моделирование (скорость, позиция, ускорение)
- **draw.go** - Рендеринг частиц в буфер экрана
- **particle.go** - Поведение отдельной частицы

**Физическая модель:**
- Частицы появляются вверху с скоростью в зависимости от направления
- Гравитация тянет вниз (VelY увеличивается каждый кадр)
- Горизонтальная скорость (VelX) имитирует ветер/боковое движение
- Длина растёт со временем (визуальный эффект)
- Обнаружение столкновений удаляет частицы в нижней части

#### **4. Анализатор звука (`audio/`)**
Захват звука в реальном времени и анализ частот.

- **analyzer.go** - интеграция с аудиомонитором и FFT анализ
- Захватывает PCM-звук на 11025Hz через команду `parec`
- Выполняет анализ частот для выделения трёх полос:
  - **Низкие** (60-180Hz): Басс, барабаны
  - **Средние** (500-2000Hz): Вокал, гитара
  - **Высокие** (2800-5000Hz): Цимбалы, верхние частоты
- Вычисляет **Огибающую** (энергия RMS) для общей динамики
- Возвращает нормализованные значения 0-1 для плавного управления параметрами

#### **5. Данные о музыке (`music/`)**
Получает информацию о трактах из различных источников.

- **track.go** - Основная структура TrackInfo и кеширование
- **playerctl.go** - Интеграция совместимого с MPRIS плеера через DBus
- **json_default.go** - Поддержка резервного JSON файла
- **artwork_path.go** - Разрешение пути к обложке
- **artwork_lookup.go** - Поиск обложки для текущего трека
- **format.go** - Форматирование информации о треке
- **mpd.go** - Поддержка MPD (заполнитель)

**Приоритет источников:**
1. Playerctl (синхронизация в реальном времени с активным MPRIS плеером)
2. JSON файл (`/tmp/virga-player/track.json`)
3. Резервный режим (без информации о треке)

#### **6. Отображение обложки (`app/artwork/`)**
Рендеринг обложки альбома с графикой терминала или текстовым резервным режимом.

- **artwork.go** - Основное состояние обложки и жизненный цикл
- **draw.go** - Координация рендеринга
- **image_io.go** - Загрузка изображений и работа с файлами
- **image_render.go** - Преобразование изображения в формат терминала
- **render_sixel.go** - Кодирование формата графики Sixel
- **render_text.go** - Текстовый/Unicode резервный рендеринг
- **sixel_support.go** - Обнаружение возможностей терминала

**Стратегия рендеринга:**
- Обнаруживает поддержку Sixel через запросы к терминалу
- Преобразует PNG в графику Sixel 256×256 если поддерживается
- Переходит на цветный текстовый макет если нет
- Применяет управляемые звуком анимации (затухание, пульс, мигание)

#### **7. Рендеринг терминала (`renderer/`)**
Низкоуровневая абстракция tcell и управление экраном.

- **renderer.go** - Управление буфером кадра и конвейер рендеринга

#### **8. Настройки (`settings/`)**
Управление конфигурацией и интерфейсом настроек.

- **config.go** - Загрузка/сохранение конфигурации (JSON)
- **theme.go** - Управление темой и цветами
- **page.go** - Абстракция страницы настроек
- **page/handler.go** - Обработчики элементов меню
- **page/menu.go** - Рендеринг и навигация меню
- **page/page.go** - Структура и макет страницы
- **page/render.go** - Логика рендеринга UI
- **theme/defaults.go** - Цвета темы по умолчанию
- **theme/loader.go** - Загрузка файла темы
- **theme/parser.go** - Парсинг JSON темы
- **theme/theme.go** - Применение темы

**Расположение конфигурации:** `~/.config/virga-player/config.json`

**Доступные настройки:**
- FPS (по умолчанию: 60)
- Максимум частиц (по умолчанию: 220)
- Значение скорости дождя (по умолчанию: 100)
- Переключатель включения дождя
- Переключатель режима реактивности на музыку
- Значение интенсивности музыки (по умолчанию: 100)
- Направление дождя (right-to-left, left-to-right, straight, random)
- Включение/отключение анимации обложки
- Переключатель UI плеера

#### **9. Сцена (`scene/`)**
Рендеринг фона и визуальная композиция.

- **scene.go** - Оркестрировка и рендеринг сцены
- Рендеринг статического городского фона (здания, улица)
- Наложение эффекта дождя из частиц
- Управление позиционированием отображения обложки
- Координация многоуровневой композиции

## Начало работы

### Требования

- **Go** 1.25 или выше
- **Linux** система
- локальный аудиобэкенд (для анализа звука)
- **ImageMagick** (команда `convert`) - для рендеринга обложек в Sixel
- **Терминал** с поддержкой 24-битного цвета (поддержка Sixel опциональна, но рекомендуется)

### Установка и сброка

```bash
chmod +x install.sh build.sh
sudo ./install.sh ; ./build.sh
```

## Конфигурация

Конфигурация хранится в `~/.config/virga-player/config.json`:

```json
{
  "fps": 60,
  "max_particles": 220,
  "rain_speed": 100,
  "rain_enabled": true,
  "music_reactive": false,
  "music_reactive_intensity": 100,
  "cover_animation": false,
  "direction": "random",
  "player": false
}
```

### Параметры конфигурации

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|---------|
| `fps` | int | 60 | Кадры в секунду (15-120) |
| `max_particles` | int | 220 | Максимум частиц в системе (20-500) |
| `rain_speed` | int | 100 | Базовое значение скорости дождя (25-300) |
| `rain_enabled` | bool | true | Включить или отключить анимацию дождя |
| `music_reactive` | bool | false | Включить реактивность частиц на музыку |
| `music_reactive_intensity` | int | 100 | Интенсивность реакции на музыку (20-200) |
| `direction` | string | "random" | Направление дождя (`right-to-left`, `left-to-right`, `straight`, `random`) |
| `cover_animation` | bool | false | Включить анимацию обложки |
| `player` | bool | false | Показать виджет информации плеера |

### Тема

Тема хранится в виде CSS в файле `~/.config/virga-player/style.css`.
Файл автоматически создаётся при первом запуске приложения.

Тема задаётся через CSS-переменные, например:

```css
:root {
  --bg: transparent;
  --message-text: white;
  --track-title: white;
  --track-artist: green;
  --track-album: yellow;
  --track-time: gray;
  --rain-head: white;
  --rain-tail: gray;
}
```

## Рендеринг обложки

- Sixel: используется при поддерживаемом терминале
- Текстовый режим: fallback при отсутствии Sixel
- Требование: `convert` из ImageMagick

## Система реактивности на музыку

- Анализ звука через локальный аудиобэкенд
- Три частотные полосы: 60-180 Hz, 500-2000 Hz, 2800-5000 Hz
- Вычисление RMS-огибающей
- При `musicReactive` параметры частиц масштабируются по анализу

## Характеристики производительности

- **FPS**: Настраивается 15-120 fps (по умолчанию 60)
- **Память**: ~20MB 
- **CPU**: ~0.8%

## Решение проблем

### Отсутствует реакция на звук

1. Проверьте состояние аудиосистемы: `pactl info`
2. Убедитесь, что источник звука активен
3. Проверьте, что `musicReactive` включен в настройках
4. Перезапустите приложение, чтобы переинициализировать анализатор звука

### Обложка не отображается

1. Проверьте `artwork_url` в источнике музыки (JSON или плеер)
2. Проверьте установлен ли ImageMagick: `which convert`
3. Для Sixel: подтвердите поддержку терминала (попробуйте `echo -e '\x1b]...` команды)
4. Проверьте разрешения на файлы обложек

### Проблемы с рендерингом терминала

1. Убедитесь в поддержке 24-битного цвета: `echo $COLORTERM`
2. Попробуйте другой эмулятор терминала (Kitty, WezTerm, и т.д.)
3. Обновите терминал на последнюю версию
4. Проверьте что шрифт терминала поддерживает Unicode символы

### Ошибки аудиосистемы

Если анализатор звука не работает:
1. Проверьте состояние аудиосистемы
2. Проверьте что пользователь в группе audio: `groups $USER`
3. Проверьте существование стандартного sink: `pactl list short sinks`

## Разработка

### Зависимости

- **tcell/v2** (github.com/gdamore/tcell/v2) - Библиотека UI терминала
- **golang.org/x/image** - Обработка изображений

Все зависимости совместимы с Go стандартной библиотекой, не требуют C/C++.

## Переменные окружения

- `COLORTERM` - Установите на `truecolor` для поддержки 24-битного цвета
- `TERM` - Тип терминала (например, `xterm-256color`, `xterm-kitty`)
- `HOME` - Используется для директории конфига `$HOME/.config/virga-player/`
